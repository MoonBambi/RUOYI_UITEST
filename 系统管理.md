# 若依系统管理模块核心场景化测试用例

## 1. 系统管理

**系统管理**是若依后台的核心，负责整个系统的权限分配、组织架构定义及安全参数配置。场景化用例旨在模拟真实业务流，验证多个功能点之间的联动性。

---

## 2. 场景化用例详情

### 场景一：新员工入职（用户-部门-角色权限联动）

**描述**：模拟企业新员工入职时，管理员为其分配部门、设定角色并验证权限。

| **测试步骤** | **操作描述**                            | **预期结果**                 |
| -------- | ----------------------------------- | ------------------------ |
| 1. 部门管理  | 创建一个新部门（如：测试部）                      | 部门树成功显示“测试部”，状态为正常       |
| 2. 角色管理  | 新增角色“软件测试员”，分配“系统管理-用户管理”菜单权限       | 角色创建成功，权限勾选已保存           |
| 3. 用户管理  | 新增用户“test_user”，选择“测试部”，关联“软件测试员”角色 | 用户保存成功，所属部门和角色显示正确       |
| 4. 权限验证  | 使用 `test_user` 账号登录系统               | 侧边栏仅显示”实例演示”，无法看到其他未授权模块 |

**测试重点**：数据的完整性、部门层级关系的展示、以及后端数据权限（Data Permission）的隔离。

---

### 场景二：安全合规控制（密码强度策略）

**描述**：验证系统对安全风险的防御能力，确保安全参数配置能实时生效。

| **测试步骤**  | **操作描述**                               | **预期结果**               |
| --------- | -------------------------------------- | ---------------------- |
| 1. 设置密码强度 | 在“参数设置”中修改 `sys.account.chrtype` 为 `1` | 修改成功                   |
| 2. 创建新用户  | 用户管理界面创建新用户，密码输入非纯数字                   | 创建用户失败，提示密码必须为纯数字（0-9） |
| 3. 创建新用户  | 用户管理界面创建新用户，密码输入纯数字                    | 创建用户成功                 |

**测试重点**：全局配置参数的实时生效性、Redis 缓存更新逻辑。
